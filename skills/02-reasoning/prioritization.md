# Prioritization

**Category:** `reasoning`  
**Skill Level:** `intermediate`  
**Stability:** `stable`
**Added:** 2025-03

### Description

Rank tasks, goals, or options by importance, urgency, and impact to determine what to work on first.

### Example

```
Tasks:
  - Fix critical auth bug    → urgent + high impact
  - Write API documentation  → low urgency + high long-term value
  - Add dark mode            → low urgency + low impact

Priority: auth bug → docs → dark mode
```
