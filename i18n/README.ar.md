## المشكلة

كل مطوِّر لوكيل ذكاء اصطناعي يُعيد اكتشاف المهارات نفسها من الصفر.

أحدهم يتعلَّم RAG بالطريقة الصعبة. وآخر يكتشف حقن الذاكرة في الساعة الثانية صباحًا. وثالث يقضي أسبوعًا كاملًا في مقارنة ReAct بـ LATS دون أن يُشارك النتائج. ورابع يصطدم بنفس أوجه الفشل التي واجهتها أنت الشهر الماضي.

**هذه المعرفة الجماعية تختفي في محادثات Slack والمستودعات الخاصة وإشارات Twitter المرجعية.**

شجرة المهارات تُصلح هذا.

---

## ما هي شجرة المهارات؟

**شجرة المهارات هي نظام التشغيل المشترك لقدرات وكيل الذكاء الاصطناعي.**

فهرس حيّ وذو نسخ ومدعوم من المجتمع لكل ما يمكن لوكيل فعله — موثَّق بكود عملي ومعايير مرجعية حقيقية وأوجه فشل وتاريخ تطوُّر. كل مهارة جاهزة للإنتاج. كل نظام يُظهر كيفية دمج المهارات. كل معيار مرجعي قابل للتكرار.

إنها ليست قائمة. إنها بنية تحتية.

git clone https://github.com/SamoTech/skills-tree.git

# البحث عن مهارة بالكلمات المفتاحية

# قراءة نظام كامل من البداية إلى النهاية

## 🤝 كيف تساهم

| النوع | ما هو | صيغة عنوان PR |
|---|---|---|
| **مهارة جديدة** | قدرة غير مُفهرَسة بعد | `feat: add [skill] to [category]` |
| **ترقية مهارة** | رفع v1 إلى v2 بمحتوى أفضل | `improve: [skill] — v1→v2` |
| **معيار مرجعي** | مقارنة مباشرة بأرقام حقيقية | `benchmark: [skill-a] vs [skill-b]` |
| **نظام / مخطط** | سير عمل أو بنية متعددة المهارات | `system: add [name]` |
